## 1. ACM 인증서 발급 ❌

### 현재 상태
- **구현**: 없음
- **배포**: 없음

### 필요한 작업

#### Step 1: Route 53 도메인 등록
```bash
# 도메인 예시: say2-6team.com 또는 api.say2-6team.com
# AWS Console → Route 53 → Register Domain
```

#### Step 2: ACM 인증서 요청
```bash
aws acm request-certificate \
  --domain-name api.say2-6team.com \
  --validation-method DNS \
  --region ap-northeast-2
```

#### Step 3: DNS 검증
- ACM Console에서 CNAME 레코드 확인
- Route 53에 CNAME 레코드 추가
- 검증 완료 대기 (5-30분)

#### Step 4: CloudFormation 템플릿 수정
`infra/compute-stack.yaml`에 파라미터 추가:
```yaml
Parameters:
  CertificateArn:
    Type: String
    Description: ACM Certificate ARN for HTTPS
    Default: ""
```

## 2. ALB HTTPS Listener ❌

### 현재 상태
- **구현**: HTTP Listener만 있음 (Port 80)
- **배포**: HTTP만 작동

### 현재 코드 (compute-stack.yaml)
```yaml
HTTPListener:
  Type: AWS::ElasticLoadBalancingV2::Listener
  Properties:
    LoadBalancerArn: !Ref ApplicationLoadBalancer
    Port: 80
    Protocol: HTTP
    DefaultActions:
      - Type: forward
        TargetGroupArn: !Ref OrchestratorTargetGroup
```

### 필요한 코드 추가
```yaml
# HTTPS Listener 추가
HTTPSListener:
  Type: AWS::ElasticLoadBalancingV2::Listener
  Properties:
    LoadBalancerArn: !Ref ApplicationLoadBalancer
    Port: 443
    Protocol: HTTPS
    Certificates:
      - CertificateArn: !Ref CertificateArn
    DefaultActions:
      - Type: forward
        TargetGroupArn: !Ref OrchestratorTargetGroup

# Listener Rules도 HTTPSListener로 변경
OrchestratorListenerRule:
  Type: AWS::ElasticLoadBalancingV2::ListenerRule
  Properties:
    ListenerArn: !Ref HTTPSListener  # 변경
    Priority: 10
    Conditions:
      - Field: path-pattern
        Values:
          - /orchestrator/*
    Actions:
      - Type: forward
        TargetGroupArn: !Ref OrchestratorTargetGroup
```

## 3. HTTP → HTTPS 리다이렉트 ❌

### 현재 상태
- **구현**: 없음
- **배포**: HTTP만 허용

### 필요한 코드 추가
```yaml
# HTTP Listener를 HTTPS로 리다이렉트
HTTPListener:
  Type: AWS::ElasticLoadBalancingV2::Listener
  Properties:
    LoadBalancerArn: !Ref ApplicationLoadBalancer
    Port: 80
    Protocol: HTTP
    DefaultActions:
      - Type: redirect
        RedirectConfig:
          Protocol: HTTPS
          Port: 443
          StatusCode: HTTP_301
```

### Security Group 수정
```yaml
# ALB Security Group에 HTTP 규칙 유지 (리다이렉트용)
ALBIngressHTTP:
  Type: AWS::EC2::SecurityGroupIngress
  Properties:
    GroupId: !Ref ALBSecurityGroup
    IpProtocol: tcp
    FromPort: 80
    ToPort: 80
    CidrIp: 0.0.0.0/0
    Description: Allow HTTP for redirect to HTTPS

# HTTPS 규칙은 이미 있음
ALBIngressHTTPS:
  Type: AWS::EC2::SecurityGroupIngress
  Properties:
    GroupId: !Ref ALBSecurityGroup
    IpProtocol: tcp
    FromPort: 443
    ToPort: 443
    CidrIp: 0.0.0.0/0
    Description: Allow HTTPS to ALB
```

## 4. WAF 규칙 ✅ (부분 구현)

### 현재 상태
- **구현**: ✅ 완료
- **배포**: ✅ 완료
- **모드**: ⚠️ **Count 모드** (차단하지 않고 관찰만 함)

### 배포된 WAF 정보
```json
{
  "Name": "say2-6team-alb-waf",
  "Id": "8e27d90b-07ba-4683-928a-c7ddae65e0ea",
  "ARN": "arn:aws:wafv2:ap-northeast-2:666803869796:regional/webacl/say2-6team-alb-waf/8e27d90b-07ba-4683-928a-c7ddae65e0ea"
}
```

### 구현된 WAF 규칙 (security-stack.yaml)

#### 1. AWS Managed Rules - Core Rule Set
```yaml
- Name: AWSManagedRulesCommonRuleSet
  Priority: 10
  OverrideAction:
    Count: {}  # ⚠️ Count 모드
  Statement:
    ManagedRuleGroupStatement:
      VendorName: AWS
      Name: AWSManagedRulesCommonRuleSet
```

**보호 기능**:
- SQL Injection 방어
- XSS (Cross-Site Scripting) 방어
- Local File Inclusion (LFI) 방어
- Remote File Inclusion (RFI) 방어

#### 2. AWS Managed Rules - Known Bad Inputs
```yaml
- Name: AWSManagedRulesKnownBadInputsRuleSet
  Priority: 20
  OverrideAction:
    Count: {}  # ⚠️ Count 모드
  Statement:
    ManagedRuleGroupStatement:
      VendorName: AWS
      Name: AWSManagedRulesKnownBadInputsRuleSet
```

**보호 기능**:
- 알려진 악성 입력 패턴 차단
- CVE 기반 공격 패턴 차단

#### 3. AWS Managed Rules - IP Reputation List
```yaml
- Name: AWSManagedRulesAmazonIpReputationList
  Priority: 30
  OverrideAction:
    Count: {}  # ⚠️ Count 모드
  Statement:
    ManagedRuleGroupStatement:
      VendorName: AWS
      Name: AWSManagedRulesAmazonIpReputationList
```

**보호 기능**:
- 악성 IP 주소 차단
- 봇넷 IP 차단
- 스캐너 IP 차단

#### 4. Rate Limiting
```yaml
- Name: RateLimitPerIP
  Priority: 100
  Action:
    Count: {}  # ⚠️ Count 모드
  Statement:
    RateBasedStatement:
      Limit: 2000  # 5분당 2000 요청
      AggregateKeyType: IP
```

**보호 기능**:
- DDoS 방어
- Brute Force 공격 방어
- API 남용 방지

### ⚠️ 주의사항

**현재 모든 규칙이 Count 모드**:
- 공격을 **차단하지 않음**
- CloudWatch Metrics에 **기록만** 함
- 프로덕션 전환 시 **Block 모드로 변경 필요**

### Block 모드로 전환 방법

#### 방법 1: CloudFormation 템플릿 수정
`AWS/security-stack.yaml` 수정:
```yaml
# Count: {} 를 None: {} 로 변경
- Name: AWSManagedRulesCommonRuleSet
  Priority: 10
  OverrideAction:
    None: {}  # Block 모드
  Statement:
    ManagedRuleGroupStatement:
      VendorName: AWS
      Name: AWSManagedRulesCommonRuleSet
```

#### 방법 2: AWS Console에서 수정
1. **WAF & Shield** 콘솔로 이동
2. **Web ACLs** → `say2-6team-alb-waf` 선택
3. **Rules** 탭
4. 각 규칙 편집:
   - `Override rule group action` → `None` 선택
   - 또는 `Rule action` → `Block` 선택

#### 방법 3: AWS CLI로 수정
```bash
# WAF 규칙 업데이트 (복잡함 - Console 권장)
aws wafv2 update-web-acl \
  --scope REGIONAL \
  --id 8e27d90b-07ba-4683-928a-c7ddae65e0ea \
  --region ap-northeast-2 \
  --lock-token <LOCK_TOKEN> \
  --rules file://waf-rules-block-mode.json
```

## 🔧 즉시 적용 가능한 개선사항

### 1. HTTP Port 80 허용 (긴급)
**현재 문제**: ALB Security Group에 HTTP 규칙 없음

**해결**: 보안팀에 요청
```bash
aws ec2 authorize-security-group-ingress \
  --group-id sg-0d702017416dadb66 \
  --protocol tcp \
  --port 80 \
  --cidr 0.0.0.0/0 \
  --region ap-northeast-2
```

### 2. WAF 모니터링 설정
**CloudWatch Dashboard 생성**:
```bash
# WAF Metrics 확인
aws cloudwatch get-metric-statistics \
  --namespace AWS/WAFV2 \
  --metric-name AllowedRequests \
  --dimensions Name=WebACL,Value=say2-6team-alb-waf Name=Region,Value=ap-northeast-2 Name=Rule,Value=ALL \
  --start-time 2026-05-17T00:00:00Z \
  --end-time 2026-05-18T23:59:59Z \
  --period 3600 \
  --statistics Sum \
  --region ap-northeast-2
```

### 3. CloudWatch Alarms 설정
```yaml
# monitoring-stack.yaml에 추가
WAFBlockedRequestsAlarm:
  Type: AWS::CloudWatch::Alarm
  Properties:
    AlarmName: !Sub "${ProjectName}-waf-blocked-requests"
    MetricName: BlockedRequests
    Namespace: AWS/WAFV2
    Statistic: Sum
    Period: 300
    EvaluationPeriods: 1
    Threshold: 100
    ComparisonOperator: GreaterThanThreshold
```
