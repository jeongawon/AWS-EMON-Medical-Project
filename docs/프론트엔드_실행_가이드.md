# 프론트엔드 실행 가이드 - ECS 백엔드 연동

## 3. 백엔드 연결 설정

### 3.1 환경 변수 확인

`.env.local` 파일이 자동으로 생성되어 있습니다:

```bash
# 파일 내용 확인
cat .env.local
```

**내용**:
```env
# ECS 백엔드 ALB DNS
VITE_BACKEND_URL=http://say2-6team-alb-698170641.ap-northeast-2.elb.amazonaws.com

# 로컬 백엔드로 전환하려면 아래 주석 해제
# VITE_BACKEND_URL=http://localhost:8000
```

### 3.2 백엔드 Health Check 테스트

프론트엔드를 실행하기 전에 백엔드가 정상인지 확인:

```bash
# ALB DNS 변수 설정
ALB_DNS="say2-6team-alb-698170641.ap-northeast-2.elb.amazonaws.com"

# Orchestrator Health Check
curl http://${ALB_DNS}/orchestrator/health

# 정상 응답 예시:
# {"status": "healthy", "service": "orchestrator", "timestamp": "2026-05-18T..."}
```

**응답이 없거나 에러가 발생하면**:
- ECS Service가 정상 실행 중인지 확인
- Target Group Health 확인
- [ECS 배포 가이드 5번](./ECS_배포_가이드_5_확인및문제해결.md) 참고

### 5.2 API 프록시 설정

`vite.config.ts`에서 백엔드 API 프록시 설정:

```typescript
server: {
  port: 3000,
  proxy: {
    "/orchestrator": {
      target: "http://say2-6team-alb-698170641.ap-northeast-2.elb.amazonaws.com",
      changeOrigin: true,
    },
    // ... 기타 엔드포인트
  },

```

**동작 방식**:
```
브라우저 → http://localhost:3000/orchestrator/health
         ↓ (Vite 프록시)
         → http://say2-6team-alb-698170641.ap-northeast-2.elb.amazonaws.com/orchestrator/health
```

## 6. 백엔드 API 테스트

### 6.1 브라우저 개발자 도구에서 확인

1. 브라우저에서 `F12` 키 (개발자 도구 열기)
2. **Console** 탭 선택
3. 다음 명령어 입력:

```javascript
// Orchestrator Health Check
fetch("/orchestrator/health")
  .then((res) => res.json())
  .then((data) => console.log(data));

// 출력 예시:
// {status: "healthy", service: "orchestrator", timestamp: "2026-05-18T..."}
```

### 6.2 Network 탭에서 확인

1. **Network** 탭 선택
2. 페이지 새로고침 (`Ctrl+R` 또는 `Cmd+R`)
3. `health` 요청 클릭
4. **Headers** 탭에서 요청/응답 확인:
   - Request URL: `http://localhost:3000/orchestrator/health`
   - Status Code: `200 OK`
   - Response: JSON 데이터

## 7. 문제 해결

### 문제 1: npm install 실패

**증상**:
```
npm ERR! code ERESOLVE
npm ERR! ERESOLVE unable to resolve dependency tree
```

**해결 방법**:
```bash
# 캐시 삭제
npm cache clean --force

# node_modules 삭제 후 재설치
rm -rf node_modules package-lock.json
npm install
```

### 문제 2: 백엔드 연결 실패

**증상**:
- 브라우저에 "❌ 연결 실패: Failed to fetch" 표시

**원인 1**: ECS Service가 실행되지 않음
```bash
# Service 상태 확인
aws ecs describe-services \
  --cluster say2-6team-ecs-cluster \
  --services say2-6team-orchestrator-service \
  --region ap-northeast-2 \
  --query 'services[0].[serviceName,status,runningCount,desiredCount]'
```

**원인 2**: Target이 unhealthy 상태
```bash
# Target Health 확인
aws elbv2 describe-target-health \
  --target-group-arn <TG_ARN> \
  --region ap-northeast-2
```

**원인 3**: CORS 문제
- 백엔드에서 CORS 설정 필요
- Orchestrator 코드에서 CORS 허용 확인

### 문제 3: 포트 3000이 이미 사용 중

**증상**:
```
Error: Port 3000 is already in use
```

**해결 방법**:
```bash
# 다른 포트 사용
npm run dev -- --port 3001

# 또는 vite.config.ts에서 포트 변경
server: {
  port: 3001,
  // ...
}
```

### 문제 4: 프록시가 작동하지 않음

**증상**:
- API 요청이 404 Not Found

**해결 방법**:
```bash
# 1. .env.local 파일 확인
cat .env.local

# 2. 개발 서버 재시작
# Ctrl+C로 중단 후
npm run dev

# 3. 브라우저 캐시 삭제
# Ctrl+Shift+Delete (Chrome)
```
