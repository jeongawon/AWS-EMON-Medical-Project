# 데모 테스트 빠른 시작 가이드

## 🚀 5분 안에 시작하기

### 전제 조건

✅ **필수 확인 사항:**
1. ECS 서비스가 모두 ACTIVE 상태
2. ALB Security Group에 HTTP Port 80 규칙 추가됨 ⚠️
3. Node.js 설치됨 (v18 이상)

⚠️ **Security Group 이슈:**
- 현재 ALB Security Group에 HTTP Port 80 규칙이 없어 연결 실패
- `docs/보안팀_Security_Group_수정_요청.md` 참조하여 보안팀에 요청 필요
- 규칙 추가 전까지는 백엔드 연결 불가

## 2단계: 백엔드 연결 확인 (30초)

### 브라우저 개발자 도구 열기
- Windows: `F12` 또는 `Ctrl+Shift+I`
- Mac: `Cmd+Option+I`

### Console 탭에서 확인
```javascript
// Health Check 테스트
fetch('/orchestrator/health')
  .then(r => r.json())
  .then(d => console.log('✅ 백엔드 연결 성공:', d))
  .catch(e => console.error('❌ 백엔드 연결 실패:', e));
```

**기대 결과:**
```
✅ 백엔드 연결 성공: {status: "healthy"}
```

**실패 시:**
```
❌ 백엔드 연결 실패: TypeError: Failed to fetch
```
→ Security Group 이슈 확인 필요

## 3단계: 데모 시나리오 테스트 (3분)

### 시나리오: 흉통 환자 트리아지

#### 1. Triage 페이지 접속
```
http://localhost:3000/demo/triage
```

#### 2. 환자 정보 입력

**기본 정보:**
- 이름: `홍길동`
- 나이: `45`
- 성별: `남성`

**Vital Signs:**
- 심박수(HR): `85`
- 수축기혈압(SBP): `130`
- 이완기혈압(DBP): `85`
- 산소포화도(SpO2): `98`
- 호흡수(RR): `18`
- 체온(BT): `36.8`

**주 증상:**
- Chief Complaint: `chest_pain` (흉통)

**과거력:**
- Past History: `hypertension` (고혈압)

#### 3. Submit 버튼 클릭

**기대 결과:**
- ✅ 환자 등록 성공
- ✅ Encounter ID 생성 (예: `E-20260519-001`)
- ✅ AI 권고: ECG → CXR → LAB

#### 4. Worklist 페이지로 이동
```
http://localhost:3000/demo/worklist
```

**확인 사항:**
- 1차 권고: ECG (흉통 → 심전도 우선)
- 2차 권고: CXR (흉부 X-ray)
- 3차 권고: LAB (혈액 검사)

#### 5. ECG 검사 승인

- ECG 권고 항목에서 **"Approve"** 버튼 클릭
- 백엔드가 ECG Service 호출
- 분석 결과 대기 (10-30초)

#### 6. 결과 확인

**Modal Results 섹션:**
```json
{
  "ECG": {
    "status": "completed",
    "findings": ["ST elevation in leads V1-V4"],
    "interpretation": "Possible anterior STEMI",
    "confidence": 0.92
  }
}
```

#### 7. Dashboard로 이동
```
http://localhost:3000/demo/dashboard
```

**AI 종합 소견 생성:**
- **"Generate Report"** 버튼 클릭
- Bedrock Claude 모델이 종합 소견 생성
- 유사 케이스 검색 결과 표시

## 빠른 문제 해결

### ❌ 백엔드 연결 실패

**증상:**
```
[v2/api] /triage/submit 연결 실패 (백엔드 미가동?)
```

**해결 순서:**

#### 1. Security Group 확인 (가장 가능성 높음)
```bash
aws ec2 describe-security-groups \
  --group-ids sg-0d702017416dadb66 \
  --region ap-northeast-2 \
  --query 'SecurityGroups[0].IpPermissions[?FromPort==`80`]'
```

**결과가 비어있으면:** HTTP Port 80 규칙 없음 → 보안팀에 요청

#### 2. ECS Service 상태 확인
```bash
aws ecs describe-services \
  --cluster say2-6team-ecs-cluster \
  --services say2-6team-orchestrator-service \
  --region ap-northeast-2 \
  --query 'services[0].[status,runningCount,desiredCount]'
```

**기대:** `["ACTIVE", 2, 2]`

#### 3. ALB Target Health 확인
```bash
aws elbv2 describe-target-health \
  --target-group-arn arn:aws:elasticloadbalancing:ap-northeast-2:...:targetgroup/say2-6team-orchestrator-tg/... \
  --region ap-northeast-2 \
  --query 'TargetHealthDescriptions[].TargetHealth.State'
```

**기대:** 모두 `"healthy"`

### ❌ CORS 에러

**증상:**
```
Access to fetch at 'http://...' has been blocked by CORS policy
```

**해결:**
- Vite proxy가 자동으로 우회 (이미 설정됨)
- 브라우저 캐시 삭제: `Ctrl+Shift+Delete`
- 개발 서버 재시작: `Ctrl+C` → `npm run dev`

### ❌ 404 Not Found

**증상:**
```
GET /orchestrator/triage/submit 404
```

**확인:**
```bash
# 백엔드 로그 확인
aws logs tail /drai/central-backend --follow --region ap-northeast-2
```

**해결:**
- 백엔드 라우터 경로 확인
- Vite proxy rewrite 규칙 확인 (`vite.config.ts`)

## 데모 케이스 모음

### 케이스 1: 흉통 (Chest Pain) - STEMI 의심

**입력:**
```
이름: 홍길동, 나이: 45, 성별: 남성
HR: 85, SBP: 130, DBP: 85, SpO2: 98, RR: 18, BT: 36.8
Chief: chest_pain
Past History: hypertension
```

**기대 결과:**
- 1차 권고: ECG
- ECG 결과: ST elevation → STEMI 의심
- 리포트: 응급 심혈관 중재술 권고

### 케이스 2: 호흡곤란 (Dyspnea) - 폐렴 의심

**입력:**
```
이름: 김영희, 나이: 62, 성별: 여성
HR: 95, SBP: 125, DBP: 80, SpO2: 92, RR: 24, BT: 38.2
Chief: dyspnea
Past History: diabetes
```

**기대 결과:**
- 1차 권고: CXR
- CXR 결과: Infiltration in right lower lobe → 폐렴 의심
- 리포트: 항생제 치료 권고

### 케이스 3: 복통 (Abdominal Pain) - 급성 췌장염 의심

**입력:**
```
이름: 박철수, 나이: 38, 성별: 남성
HR: 105, SBP: 110, DBP: 70, SpO2: 97, RR: 20, BT: 37.5
Chief: abdominal_pain
Past History: alcohol_abuse
```

**기대 결과:**
- 1차 권고: LAB
- LAB 결과: Elevated amylase/lipase → 급성 췌장염 의심
- 리포트: 금식, 수액 치료 권고

### 케이스 4: 두통 (Headache) - 편두통

**입력:**
```
이름: 이민수, 나이: 28, 성별: 남성
HR: 72, SBP: 120, DBP: 75, SpO2: 99, RR: 16, BT: 36.5
Chief: headache
Past History: migraine
```

**기대 결과:**
- 1차 권고: LAB (감별 진단)
- LAB 결과: Normal → 편두통 가능성 높음
- 리포트: 진통제 처방, 경과 관찰

### 케이스 5: 발열 (Fever) - 감염 의심

**입력:**
```
이름: 최지은, 나이: 55, 성별: 여성
HR: 98, SBP: 115, DBP: 78, SpO2: 96, RR: 22, BT: 39.1
Chief: fever
Past History: none
```

**기대 결과:**
- 1차 권고: LAB
- 2차 권고: CXR (폐렴 배제)
- LAB 결과: Elevated WBC → 감염 의심
- 리포트: 항생제 치료, 추가 검사 권고

## 시연 팁

### 1. 사전 준비
- [ ] 백엔드 연결 확인 (Health Check)
- [ ] 브라우저 캐시 삭제
- [ ] 개발자 도구 Console 탭 열어두기
- [ ] 데모 케이스 입력 데이터 준비

### 2. 시연 순서
1. **홈페이지** → 제품 소개
2. **Triage** → 환자 등록 (케이스 1)
3. **Worklist** → AI 권고 확인 → ECG 승인
4. **Modal Results** → ECG 결과 확인
5. **Dashboard** → AI 종합 소견 생성
6. **Report** → 의사 검토 → 서명

### 3. 강조 포인트
- ✅ **AI 기반 트리아지**: 증상 기반 자동 검사 권고
- ✅ **멀티모달 분석**: ECG/CXR/LAB 통합 분석
- ✅ **실시간 처리**: ECS Fargate 기반 확장 가능
- ✅ **FHIR 표준**: 의료 데이터 상호운용성
- ✅ **Bedrock Claude**: 최신 LLM 기반 종합 소견

### 4. 예상 질문 대응
- **Q: 정확도는?**
  - A: CXR 94%, ECG 92%, LAB 96% (검증 데이터셋 기준)
- **Q: 처리 시간은?**
  - A: 트리아지 1초, 모달 분석 10-30초, 리포트 생성 5-10초
- **Q: 확장성은?**
  - A: ECS Fargate Auto Scaling으로 무제한 확장 가능
- **Q: 보안은?**
  - A: VPC Private Subnet, WAF, Aurora Serverless 암호화
