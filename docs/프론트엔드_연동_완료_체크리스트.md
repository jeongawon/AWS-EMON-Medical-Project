## 🔍 문제 발생 시 대응

### 문제 1: Security Group 수정 후에도 연결 실패

**확인 사항:**
```bash
# 1. Security Group 규칙 재확인
aws ec2 describe-security-groups \
  --group-ids sg-0d702017416dadb66 \
  --region ap-northeast-2 \
  --query 'SecurityGroups[0].IpPermissions[?FromPort==`80`]'

# 2. ALB Listener 확인
aws elbv2 describe-listeners \
  --load-balancer-arn $(aws elbv2 describe-load-balancers \
    --names say2-6team-alb \
    --region ap-northeast-2 \
    --query 'LoadBalancers[0].LoadBalancerArn' \
    --output text) \
  --region ap-northeast-2 \
  --query 'Listeners[].[Port,Protocol]'

# 3. Target Health 재확인
aws elbv2 describe-target-health \
  --target-group-arn $(aws elbv2 describe-target-groups \
    --names say2-6team-orchestrator-tg \
    --region ap-northeast-2 \
    --query 'TargetGroups[0].TargetGroupArn' \
    --output text) \
  --region ap-northeast-2
```

**해결:**
- Security Group 규칙이 제대로 추가되었는지 확인
- ALB Listener가 Port 80을 리스닝하는지 확인
- Target이 healthy 상태인지 확인

### 문제 2: 특정 API 엔드포인트만 실패

**확인 사항:**
```bash
# 백엔드 로그 확인
aws logs tail /drai/central-backend --follow --region ap-northeast-2

# 특정 엔드포인트 직접 테스트
curl -X POST http://${ALB_DNS}/orchestrator/triage/submit \
  -H "Content-Type: application/json" \
  -d '{"patient":{"name":"Test","age":30,"gender":"male"},...}'
```

**해결:**
- 백엔드 로그에서 에러 메시지 확인
- API 경로 불일치 확인 (Vite proxy rewrite vs 백엔드 라우터)
- 요청 페이로드 형식 확인

### 문제 3: 프론트엔드 빌드/실행 오류

**확인 사항:**
```bash
# Node.js 버전 확인
node --version
# 기대: v18 이상

# npm 버전 확인
npm --version
# 기대: v9 이상

# 의존성 재설치
rm -rf node_modules package-lock.json
npm install
```

**해결:**
- Node.js 버전 업그레이드 (필요 시)
- 의존성 재설치
- 캐시 삭제 후 재시도
