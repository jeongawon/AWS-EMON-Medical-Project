## 🚨 1단계: Security Group 확인

### 현재 Security Group 규칙 확인
```bash
aws ec2 describe-security-groups \
  --group-ids sg-0d702017416dadb66 \
  --region ap-northeast-2 \
  --query 'SecurityGroups[0].IpPermissions[].[IpProtocol,FromPort,ToPort,IpRanges[0].CidrIp]' \
  --output table
```

**기대 결과:**
```
---------------------------------
|  DescribeSecurityGroups       |
+------+------+------+-----------+
| tcp  | 443  | 443  | 0.0.0.0/0 |
+------+------+------+-----------+
```

**문제:** Port 80이 없음!

---

## 🔧 2단계: Security Group 수정 (보안팀 전달용)

### AWS CLI 명령어
```bash
aws ec2 authorize-security-group-ingress \
  --group-id sg-0d702017416dadb66 \
  --protocol tcp \
  --port 80 \
  --cidr 0.0.0.0/0 \
  --region ap-northeast-2
```

### AWS Console 방법
1. EC2 Console → Security Groups
2. Security Group ID: `sg-0d702017416dadb66` 검색
3. Inbound rules → Edit inbound rules
4. Add rule:
   - Type: `Custom TCP`
   - Port: `80`
   - Source: `0.0.0.0/0`
   - Description: `Allow HTTP to ALB for development/demo`
5. Save rules

## 📊 모니터링 명령어

### CloudWatch 메트릭 확인

#### ALB 요청 수
```bash
aws cloudwatch get-metric-statistics \
  --namespace AWS/ApplicationELB \
  --metric-name RequestCount \
  --dimensions Name=LoadBalancer,Value=app/say2-6team-alb/... \
  --start-time $(date -u -d '1 hour ago' +%Y-%m-%dT%H:%M:%S) \
  --end-time $(date -u +%Y-%m-%dT%H:%M:%S) \
  --period 300 \
  --statistics Sum \
  --region ap-northeast-2
```

#### ECS CPU 사용률
```bash
aws cloudwatch get-metric-statistics \
  --namespace AWS/ECS \
  --metric-name CPUUtilization \
  --dimensions Name=ServiceName,Value=say2-6team-orchestrator-service Name=ClusterName,Value=say2-6team-ecs-cluster \
  --start-time $(date -u -d '1 hour ago' +%Y-%m-%dT%H:%M:%S) \
  --end-time $(date -u +%Y-%m-%dT%H:%M:%S) \
  --period 300 \
  --statistics Average \
  --region ap-northeast-2
```
