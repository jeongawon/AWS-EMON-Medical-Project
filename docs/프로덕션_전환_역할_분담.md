## 🔴 긴급: HTTP Port 80 허용 (보안팀)

### 담당
- **부서**: 보안팀
- **담당자**: 양정인 (yji)
- **Stack**: `security-stack.yaml`

### 작업 내용
ALB Security Group에 HTTP (Port 80) 인바운드 규칙 추가

### 방법 1: AWS Console (권장)
1. **EC2 Console** → **Security Groups**
2. Security Group ID: `sg-0d702017416dadb66` 검색
3. **Inbound rules** → **Edit inbound rules**
4. **Add rule**:
   - Type: `Custom TCP`
   - Port: `80`
   - Source: `0.0.0.0/0`
   - Description: `Allow HTTP to ALB for development/demo`
5. **Save rules**

### 방법 2: CloudFormation 수정
`AWS/security-stack.yaml`에 추가:
```yaml
ALBIngressHTTP:
  Type: AWS::EC2::SecurityGroupIngress
  Properties:
    GroupId: !Ref ALBSecurityGroup
    IpProtocol: tcp
    FromPort: 80
    ToPort: 80
    CidrIp: 0.0.0.0/0
    Description: Allow HTTP to ALB for development/demo
```

그 후:
```bash
aws cloudformation update-stack \
  --stack-name say2-6team-security \
  --template-body file://AWS/security-stack.yaml \
  --capabilities CAPABILITY_IAM \
  --region ap-northeast-2
```

### 예상 소요 시간
- Console: 5분
- CloudFormation: 10분

### 우선순위
🔴 **긴급** - 프론트엔드 데모가 차단되어 있음

## 🟡 ACM 인증서 발급 (보안팀)

### 담당
- **부서**: 보안팀
- **담당자**: 양정인 (yji)
- **Stack**: `security-stack.yaml` (또는 별도 관리)

### 작업 내용
SSL/TLS 인증서 발급 (무료)

### 사전 요구사항
- Route 53 도메인 필요

### 방법 1: AWS Console (권장)
1. **ACM Console** → **Request certificate**
2. **Request a public certificate**
3. Domain name: `api.say2-6team.com` (또는 `*.say2-6team.com`)
4. Validation method: **DNS validation**
5. **Request**
6. Route 53에 CNAME 레코드 추가 (자동 또는 수동)
7. 검증 완료 대기 (5-30분)

### 방법 2: AWS CLI
```bash
# 인증서 요청
aws acm request-certificate \
  --domain-name api.say2-6team.com \
  --validation-method DNS \
  --region ap-northeast-2

# 출력된 Certificate ARN 저장
# arn:aws:acm:ap-northeast-2:666803869796:certificate/xxxxx
```

### CloudFormation 통합 (선택사항)
`AWS/security-stack.yaml`에 추가:
```yaml
Parameters:
  DomainName:
    Type: String
    Default: api.say2-6team.com
    Description: Domain name for ACM certificate

Resources:
  ACMCertificate:
    Type: AWS::CertificateManager::Certificate
    Properties:
      DomainName: !Ref DomainName
      ValidationMethod: DNS
      DomainValidationOptions:
        - DomainName: !Ref DomainName
          HostedZoneId: !Ref HostedZoneId
      Tags:
        - Key: Name
          Value: !Sub "${ProjectName}-acm-cert"
        - Key: Project
          Value: !Ref ProjectName

Outputs:
  CertificateArn:
    Description: ACM Certificate ARN
    Value: !Ref ACMCertificate
    Export:
      Name: !Sub "${ProjectName}-acm-cert-arn"
```

### 예상 소요 시간
- Console: 30분 (검증 포함)
- CloudFormation: 1시간

### 우선순위
🟡 **중간** - 도메인 등록 후 진행

## 🟡 HTTPS Listener 추가 (컴퓨팅팀)

### 담당
- **부서**: 컴퓨팅팀
- **담당자**: 이정인 (lji)
- **Stack**: `compute-stack.yaml`

### 작업 내용
ALB에 HTTPS Listener 추가 및 HTTP → HTTPS 리다이렉트 설정

### 사전 요구사항
- ACM 인증서 ARN 필요

### CloudFormation 수정
`infra/compute-stack.yaml` 수정:

#### 1. 파라미터 추가
```yaml
Parameters:
  CertificateArn:
    Type: String
    Description: ACM Certificate ARN for HTTPS
    Default: ""
```

#### 2. HTTPS Listener 추가
```yaml
HTTPSListener:
  Type: AWS::ElasticLoadBalancingV2::Listener
  Properties:
    LoadBalancerArn: !Ref ApplicationLoadBalancer
    Port: 443
    Protocol: HTTPS
    Certificates:
      - CertificateArn: !Ref CertificateArn
    DefaultActions:
      - Type: forward
        TargetGroupArn: !Ref OrchestratorTargetGroup
```

#### 3. HTTP Listener를 리다이렉트로 변경
```yaml
HTTPListener:
  Type: AWS::ElasticLoadBalancingV2::Listener
  Properties:
    LoadBalancerArn: !Ref ApplicationLoadBalancer
    Port: 80
    Protocol: HTTP
    DefaultActions:
      - Type: redirect
        RedirectConfig:
          Protocol: HTTPS
          Port: 443
          StatusCode: HTTP_301
```

#### 4. Listener Rules를 HTTPS로 변경
```yaml
OrchestratorListenerRule:
  Type: AWS::ElasticLoadBalancingV2::ListenerRule
  Properties:
    ListenerArn: !Ref HTTPSListener  # HTTP → HTTPS 변경
    Priority: 10
    Conditions:
      - Field: path-pattern
        Values:
          - /orchestrator/*
    Actions:
      - Type: forward
        TargetGroupArn: !Ref OrchestratorTargetGroup
```

### 배포
```bash
# compute-stack-params.json에 Certificate ARN 추가
# 그 후 Stack 업데이트
aws cloudformation update-stack \
  --stack-name say2-6team-compute \
  --template-body file://infra/compute-stack.yaml \
  --parameters file://infra/compute-stack-params.json \
  --capabilities CAPABILITY_IAM \
  --region ap-northeast-2
```

### 예상 소요 시간
- 코드 수정: 30분
- 배포 및 테스트: 30분
- **총 1시간**

### 우선순위
🟡 **중간** - ACM 인증서 발급 후 진행

## 🟢 WAF Block 모드 전환 (보안팀)

### 담당
- **부서**: 보안팀
- **담당자**: 양정인 (yji)
- **Stack**: `security-stack.yaml`

### 작업 내용
WAF 규칙을 Count 모드에서 Block 모드로 전환

### 사전 요구사항
- 1주일 이상 Count 모드로 관찰
- CloudWatch Metrics 확인
- False Positive 확인

### 방법 1: AWS Console (권장)
1. **WAF & Shield Console** → **Web ACLs**
2. `say2-6team-alb-waf` 선택
3. **Rules** 탭
4. 각 규칙 편집:
   - `AWSManagedRulesCommonRuleSet`
   - `AWSManagedRulesKnownBadInputsRuleSet`
   - `AWSManagedRulesAmazonIpReputationList`
   - `RateLimitPerIP`
5. `Override rule group action` → `None` 선택
6. **Save**

### 방법 2: CloudFormation 수정
`AWS/security-stack.yaml` 수정:
```yaml
# Count: {} 를 None: {} 로 변경
- Name: AWSManagedRulesCommonRuleSet
  Priority: 10
  OverrideAction:
    None: {}  # Count: {} → None: {}
  Statement:
    ManagedRuleGroupStatement:
      VendorName: AWS
      Name: AWSManagedRulesCommonRuleSet
```

### 모니터링 강화
```bash
# CloudWatch Alarm 추가
aws cloudwatch put-metric-alarm \
  --alarm-name say2-6team-waf-blocked-requests \
  --metric-name BlockedRequests \
  --namespace AWS/WAFV2 \
  --statistic Sum \
  --period 300 \
  --evaluation-periods 1 \
  --threshold 100 \
  --comparison-operator GreaterThanThreshold \
  --region ap-northeast-2
```

### 예상 소요 시간
- Console: 10분
- CloudFormation: 20분
- 모니터링 설정: 30분
- **총 1시간**

### 우선순위
🟢 **낮음** - 프로덕션 전환 시 진행 (1주일 관찰 후)
