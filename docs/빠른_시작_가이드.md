# 빠른 시작 가이드 - 프론트엔드 + ECS 백엔드

> **목표**: 5분 안에 프론트엔드를 실행하고 ECS 백엔드와 연동된 데모 확인

---

## 🚀 빠른 실행 (3단계)

### 1단계: 프론트엔드 디렉토리로 이동

**Windows (CMD 또는 PowerShell)**:
```cmd
cd final\central\frontend
```

**Mac/Linux 또는 Git Bash**:
```bash
cd final/central/frontend
```

---

### 2단계: 실행 스크립트 실행

**Windows**:
```cmd
start-frontend.bat
```

**Mac/Linux 또는 Git Bash**:
```bash
chmod +x start-frontend.sh
bash start-frontend.sh
```

---

### 3단계: 브라우저에서 확인

자동으로 브라우저가 열리거나, 수동으로 다음 주소를 열어주세요:

```
http://localhost:3000
```

**정상 화면**:
- ✅ 백엔드 정상 연결 (초록색 박스)
- 4개 서비스 카드 표시
- 배포 정보 표시

---

## 📋 실행 과정 상세

### 스크립트가 자동으로 하는 일

1. **Node.js 확인**
   - Node.js와 npm이 설치되어 있는지 확인
   - 없으면 설치 안내

2. **의존성 설치**
   - `node_modules`가 없으면 자동으로 `npm install` 실행
   - 이미 있으면 건너뜀

3. **백엔드 연결 테스트**
   - ECS ALB에 Health Check 요청
   - 연결 성공/실패 표시

4. **개발 서버 시작**
   - Vite 개발 서버 실행
   - `http://localhost:3000` 에서 접속 가능

## 🎯 데모 시나리오

### 시나리오 1: 백엔드 연결 확인

1. 브라우저에서 `http://localhost:3000` 접속
2. "백엔드 연결 상태" 섹션 확인
3. ✅ 초록색 박스가 표시되면 성공!

**표시되는 정보**:
- 서비스: orchestrator
- 상태: healthy
- 시간: 현재 시각

### 시나리오 2: 개발자 도구에서 API 확인

1. `F12` 키 눌러서 개발자 도구 열기
2. **Console** 탭 선택
3. 다음 명령어 입력:

```javascript
fetch("/orchestrator/health")
  .then(res => res.json())
  .then(data => console.log(data));
```

4. 응답 확인:
```json
{
  "status": "healthy",
  "service": "orchestrator",
  "timestamp": "2026-05-18T10:30:00Z"
}
```

### 시나리오 3: Network 탭에서 요청 확인

1. **Network** 탭 선택
2. 페이지 새로고침 (`Ctrl+R`)
3. `health` 요청 클릭
4. 요청/응답 헤더 확인:
   - Status: `200 OK`
   - Request URL: `http://localhost:3000/orchestrator/health`
   - Response: JSON 데이터

## ❌ 문제 해결

### 문제 1: Node.js가 설치되지 않음

**증상**:
```
'node'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는 배치 파일이 아닙니다.
```

**해결**:
1. https://nodejs.org/ 접속
2. LTS 버전 다운로드 (18.x 이상)
3. 설치 후 터미널 재시작
4. `node --version` 으로 확인

### 문제 2: 백엔드 연결 실패

**증상**:
- 브라우저에 "❌ 연결 실패" 표시

**해결**:
```bash
# 1. ECS Service 상태 확인
aws ecs describe-services \
  --cluster say2-6team-ecs-cluster \
  --services say2-6team-orchestrator-service \
  --region ap-northeast-2

# 2. Target Health 확인
aws elbv2 describe-target-health \
  --target-group-arn <TG_ARN> \
  --region ap-northeast-2

# 3. 로그 확인
aws logs tail /drai/central-backend --since 10m --region ap-northeast-2
```

### 문제 3: 포트 3000이 이미 사용 중

**증상**:
```
Port 3000 is already in use
```

**해결**:
```bash
# 다른 포트로 실행
npm run dev -- --port 3001

# 브라우저에서 http://localhost:3001 접속
```

### 문제 4: npm install 실패

**증상**:
```
npm ERR! code ERESOLVE
```

**해결**:
```bash
# 캐시 삭제
npm cache clean --force

# node_modules 삭제 후 재설치
rm -rf node_modules package-lock.json
npm install
```
