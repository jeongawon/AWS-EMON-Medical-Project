전체 파이프라인 흐름

🗄️

① 원본 데이터 수집 (GCS)

MIMIC-IV (환자 기본정보·입원·혈액검사·진단코드·활력징후·ECG) + MIMIC-CXR (흉부 X-ray 판독 결과)를 GCS 버킷 gs://orch-333/에 저장.

GCS 약 31 GB
⚙️

② 전처리 — 5개 Data Mart 빌드 (build_datamarts.py)

PySpark on Dataproc에서 순차적으로 5개 테이블을 생성. 소요 약 2~4시간.

① encounter_master — 방문 1건당 1행 (환자 정보+트리아지+주진단)
② modality_events — ECG·CXR·LAB 검사 이벤트 + 각 feature 49개 (이진화)
③ state_snapshots — 결과 확인 가능 시점마다 환자 상태 스냅샷 (3-State 인코딩)
④ actions_labels — 스냅샷 시점 이후 1시간 내 실제 검사 오더를 레이블로 기록
⑤ cc_prior_features — 주호소별 검사 추천 확률 (train set에서만 집계)
PySpark Parquet 저장
🔀

③ 데이터셋 조립 & 분할

5개 Data Mart를 조인 → 최종 feature 81개, 레이블 5개. subject_id 기준 70/15/15 분할로 동일 환자 데이터 유출 방지.

Train 232K / Val 50K / Test 50K
🧠

④ 모델 학습 — LightGBM 이진 분류기

Initial 모델 3개 + Follow-up 모델 5개 = 총 8개의 독립 LightGBM 이진 분류 모델. 학습 패러다임: Imitation Learning (실제 의료진 오더 패턴을 정답으로 모방).

LightGBM ~수십 분
🚀

⑤ 실시간 추론 — HybridDecisionEngine

환자 입실 → Initial 모델(첫 검사 결정) → 결과 수신 → Follow-up 모델(다음 검사/종료/에스컬레이션) → 최대 3회 반복 후 소견서 생성.

Argmax 전략 최대 3회 반복
02

3개 모달리티 전처리 방식

TAT(결과 대기시간) — 스냅샷 타이밍의 기준

각 검사의 result_available_time = 검사 시각 + TAT. 이 시점마다 상태 스냅샷을 기록.

ECG TAT: 10분
24개 이진 feature
ICD-10 코드 기반 — 심혈관 14개 (급성 심근경색·심방세동·심부전 등) + 비심혈관 10개 (당뇨·신부전·패혈증 등)

※ TAT 10분: AHA/ACC 가이드라인 기준 (MIMIC 데이터에 판독 완료 시각 없음)
CXR TAT: 30분
14개 이진 feature
CheXpert AI 판독 기반 — 무기폐·심비대·폐렴·기흉·흉수 등
불확실 소견(-1.0)은 U-Zeros 전략으로 0 처리 (False Positive 억제)

※ TAT 30분: 응급 방사선 실측 범위(20~48분) 내 보수적 중간값
LAB TAT: 70분 (실증)
10개 이진 feature
abnormal flag 기반 — Troponin T·Lactate·Creatinine·Hemoglobin·WBC 등

※ TAT 70분: labevents storetime - charttime 실측 중앙값(p50)
3-State 인코딩 — 스냅샷 시점 검사 상태
0 NOT_ORDERED — 아직 검사 오더 없음
1 PENDING — 시행했으나 결과 대기 중
2 AVAILABLE — 결과 확인 가능
03

LLM 에스컬레이션 트리거 규칙

need_reasoning = 1이 되는 조건 (규칙 기반, 학습 레이블로도 사용)
규칙 조건 임상 근거
Rule 1 급성 심근경색·심정지·폐색전증·Troponin 이상·기흉 중 하나라도 양성 즉각적 전문의 판단 필요 고위험 단일 소견
Rule 2 심방세동(ECG) 있으나 Troponin(LAB) 정상 — ECG+LAB 모두 완료 시 검사 결과 충돌 → 추가 판단 필요
Rule 3 중증도(acuity ≤ 2) + 모든 검사 완료 + 모든 결과 정상 중증 환자인데 이상 없음 → 놓친 진단 가능성
Rule 4 체류 시간 > 4시간 복잡한 케이스, 장기 대기
04

모델 구조 — Initial vs Follow-up

Initial

첫 검사 결정 모델

모델 수LightGBM × 3
출력 레이블order_ecg / order_cxr / order_lab
결정 방식3개 이진 확률의 argmax
사용 데이터첫 번째 스냅샷만 (action_index=1)
주요 feature나이·성별·주호소·활력징후·CC Prior
Overall 정확도91.33%
주의: CXR 클래스가 2.5%로 심각한 불균형 → CXR 예측 정확도 0%. 향후 개선 필요.
Follow-up

추가 검사/종료 결정 모델

모델 수LightGBM × 5
출력 레이블order_ecg / order_cxr / order_lab / stop / need_reasoning
결정 방식5개 이진 확률의 argmax
사용 데이터모든 스냅샷 (action_index ≥ 1)
추가 featurehas_ecg/cxr/lab (3-State) + ECG·CXR·LAB 결과 49개
에스컬레이션 기준신뢰도 < 0.6 또는 need_reasoning=1
05

실시간 추론 흐름 (HybridDecisionEngine)

1
환자 응급실 입실 → Initial 모델 실행
입력: 주호소·나이·바이탈·CC Prior
↓ 첫 검사 오더 (ECG / CXR / LAB 중 argmax)
2
검사 시행 → 결과 대기 (PENDING) → 결과 수신 (AVAILABLE)
↓
3
Follow-up 모델 실행 (최대 3회 반복)
입력: 기존 상태 + 검사 결과 feature 49개
출력: 다음 오더 / stop / need_reasoning 중 argmax
↓ 분기
stop = 1
검사 종료 → 소견서 자동 생성
need_reasoning = 1 또는 신뢰도 < 0.6
Bedrock LLM 에스컬레이션 (전문의 판단 요청)
